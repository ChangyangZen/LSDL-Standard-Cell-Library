# lsdl_basic/

LSDL cells with a single n-FET evaluation tree (paper Fig. 1).

Wave 0 cells:

- `lsdl_inv_x1` — single-NMOS tree. Smallest LSDL cell; Fig. 1
  validation in isolation.
- `lsdl_nand2_x1` — 2-NMOS series tree. Stack-of-2 validation +
  charge-sharing on the internal stack node.

Wave 1+ cells (planned): `lsdl_and{2,3,4}_x1`, `lsdl_or{2,3,4}_x1`,
`lsdl_aoi21_x1`, `lsdl_aoi22_x1`, `lsdl_mux2_x1`.

## Per-cell file convention

For each cell `<name>`:

- `<name>.mag` — Magic layout (authoritative source).
- `<name>.spice` — hand-source SPICE netlist with paper Fig. 1
  devices (Precharge, Foot, Predriver p/n, Header, Cut feedback,
  Feedback p/n, Output driver p/n) plus the cell-specific n-tree.
- `<name>.lef` — generated from Magic.
- `<name>.gds` — generated from Magic (build artifact).
- `<name>.lib` — Liberty, generated by `scripts/gen_liberty.py`.
- `<name>.v` — blackbox Verilog wrapper, generated.

Descriptor: `../../descriptor/<name>.yaml`.

Wave 0 deliverable. Programmatic layout via Magic TCL; see
`lsdl_inv_x1_layout.tcl` and `../../scripts/build_layout.sh`.

## DRC rules that are deferred to chip level

Two GF180MCU rules **fail in standalone cell DRC** and **pass only
when surrounded by fillers + endcaps at PnR time**:

| Rule | Meaning | Why it fails standalone |
|---|---|---|
| **DF.13_MV** | Max 15 µm from PCOMP (PMOS source/drain) to N+ tap in n-well | Functional cells have no in-cell taps |
| **DF.14_MV** | Max 15 µm from NCOMP (NMOS source/drain) to P+ tap in p-substrate | Same |

This is **standard practice** in commercial standard-cell libraries
and is how the upstream `gf180mcu_fd_sc_mcu9t5v0` cells are built:
the `__inv_1.mag`, `__nand2_1.mag`, etc. have no taps — they live
in the `__endcap.mag` and `__fill_{1,2,4,8,16,32,64}.mag` cells that
LibreLane places automatically during placement.

When running KLayout DRC on a single cell in isolation, **filter
out** DF.13_MV and DF.14_MV from the sign-off criterion. They are
re-checked at the top-level chip DRC after fillers are placed.

The DRC sign-off command for a single cell:

```bash
./lsdl_lib/scripts/run_in_container.sh "python3 \
  /soe/czeng14/software/pdk/gf180mcuD/libs.tech/klayout/drc/run_drc.py \
  --path=lsdl_inv_x1.gds --topcell=lsdl_inv_x1 \
  --variant=D --run_mode=flat --no_offgrid \
  --run_dir=/soe/czeng14/projects/brainstorm-domino-tmp/drc_runs \
  --mp=4"
```

Then post-process the `Violated rules` line: a cell is "DRC clean"
if `{DF.13_MV, DF.14_MV}` is a superset of the violated set.

# Geometry / Sizing Derivation (`lsdl_inv_x1` and family)

This section documents how the cell's transistor widths, multi-finger
counts, rail positions, and Y-coordinates were derived from the
GF180MCU 5V PDK constraints. Read this before changing any sizing or
re-placing FETs.

## Why the original sizing didn't fit

The first attempt placed `XPDRVP` and `XODRVP` as single-finger
PMOS at W = 1.5 µm and W = 1.5 µm respectively, NMOS up to W = 0.9 µm,
in the 5.04 µm 9T 5V row (SITE `GF018hv5v_green_sc9`). This produced
a layout where the PMOS **gate-contact M1 patches overlapped the
VPWR rail** (electrical short) and the cell could not pass M1.2a
spacing. The geometry was structurally invalid, not just incomplete.

The fix below derives the **maximum reasonable transistor widths**
that fit in 5.04 µm given the device-generator output and the GF180MCU
DRC rule deck.

## Per-FET internal geometry (from the GF180MCU `_draw` proc)

Measured by inspecting `pfet_W{0p5,1p0,1p5}_L0p5.mag` and
`nfet_W{0p4,0p5,0p8,0p9}_L0p6.mag` saved subcells:

| Layer | Y position relative to FET center (µm) |
|---|---|
| MV diffusion (active) top edge | `cy + W/2` |
| MV diffusion bottom edge | `cy − W/2` |
| S/D contact M1 cap top edge | `cy + W/2 − 0.01` (inside diff) |
| S/D contact M1 cap bottom edge of top cap | `cy + W/2 − 0.065` |
| **Gate-contact M1 patch top edge** | `cy + W/2 + 0.395`  (THE BINDING CONSTRAINT) |
| Gate-contact M1 patch bottom edge of top patch | `cy + W/2 + 0.165` |

The gate-contact M1 patch always extends **0.395 µm above the active
top edge** (and symmetrically below the bottom). The patch is 0.23 µm
tall. This is independent of W; only L = 0.5/0.6 affects the X-extent.

## DRC constraints that bind vertical placement

Sources: `${PDK_ROOT}/gf180mcuD/libs.tech/klayout/drc/rule_decks/*.drc`.

| Rule | Limit | Where it bites the LSDL cell |
|---|---|---|
| **M1.2a** | M1 spacing ≥ 0.23 µm | Between gate-contact M1 patches and VPWR/VGND rails |
| **M1.1** | M1 minimum width ≥ 0.23 µm | Sets minimum rail thickness |
| **DF.4c_MV** | N-well overlap of PCOMP ≥ 0.6 µm (all directions) | N-well must extend ≥ 0.6 µm beyond every PMOS diffusion edge |
| **DF.16_MV** | N-well spacing to NCOMP ≥ 0.6 µm | N-well bottom must be ≥ 0.6 µm above NMOS top diffusion |

## Vertical Y budget — the binding inequality

Let:
- `rail_h` = M1 rail thickness (≥ 0.23 µm per M1.1)
- `cy_p` = PMOS row centerline Y
- `cy_n` = NMOS row centerline Y
- `W_p, W_n` = the largest PMOS / NMOS device width in the cell
- Cell height = 5.04 µm

Then:

**(a) PMOS gate-contact M1 must clear VPWR rail by ≥ 0.23 µm (M1.2a):**
```
cy_p + W_p/2 + 0.395 + 0.23 ≤ (5.04 − rail_h)
→ cy_p ≤ 4.415 − W_p/2 − rail_h
```

**(b) NMOS gate-contact M1 must clear VGND rail by ≥ 0.23 µm (M1.2a):**
```
cy_n − W_n/2 − 0.395 − 0.23 ≥ rail_h
→ cy_n ≥ 0.625 + W_n/2 + rail_h
```

**(c) N-well must overlap PMOS bottom diff by ≥ 0.6 µm (DF.4c_MV):**
```
nwell_bottom ≤ cy_p − W_p/2 − 0.6
```

**(d) N-well must be ≥ 0.6 µm above NMOS top diff (DF.16_MV):**
```
nwell_bottom ≥ cy_n + W_n/2 + 0.6
```

Combining (c) and (d):
```
cy_p − cy_n ≥ W_p/2 + W_n/2 + 1.2
```

Combining (a) and (b):
```
cy_p − cy_n ≤ 4.415 − W_p/2 − rail_h − 0.625 − W_n/2 − rail_h
             = 3.79 − W_p/2 − W_n/2 − 2·rail_h
```

The two combined:
```
W_p/2 + W_n/2 + 1.2 ≤ cy_p − cy_n ≤ 3.79 − W_p/2 − W_n/2 − 2·rail_h
→ W_p + W_n ≤ 2.59 − 2·rail_h
```

## Two choices of rail thickness

**Choice 1 — Wide rails (rail_h = 0.46 µm, current stock-cell style):**
```
W_p + W_n ≤ 2.59 − 0.92 = 1.67 µm
```
Even W_n = 0.5 leaves only W_p ≤ 1.17 µm. **W_p = 1.5 (single finger)
doesn't fit** — must multi-finger XPDRVP and XODRVP.

**Choice 2 — Minimum-width rails (rail_h = 0.23 µm):**
```
W_p + W_n ≤ 2.59 − 0.46 = 2.13 µm
```
With W_n = 0.5: W_p ≤ 1.63 µm. **W_p = 1.5 fits** as single finger.
This is the chosen configuration.

The rail acts as a M1 abutment strip — narrower is fine functionally
because PnR places the cells side-by-side; adjacent rails merge.

## Multi-finger decision (the `nf` parameter of `*_draw`)

For each "logical" FET in the SPICE schematic, we choose how many
fingers to draw based on whether the SPICE W exceeds the per-finger
W_max budget. The Magic device generator's `nf` parameter draws N
fingers internally sharing source/drain alternately. The extracted
SPICE sees ONE device of W = nf · W_finger, which **matches the hand
SPICE without modification** — no PVT re-validation needed.

Per-FET sizing for `lsdl_inv_x1`:

| FET (SPICE) | SPICE W | Per-finger W | `nf` | Cell-row vertical fit |
|---|---|---|---|---|
| XPRE      | 1.0 | 1.0  | 1 | OK |
| XFBP      | 0.5 | 0.5  | 1 | OK |
| XCUTFB    | 0.5 | 0.5  | 1 | OK |
| **XPDRVP**| 3.0 | 0.75 | **4** | OK (effective W = 4 × 0.75 = 3.0) |
| **XODRVP**| 1.5 | 0.75 | **2** | OK (effective W = 2 × 0.75 = 1.5) |
| **XNTREE**| 0.8 | 0.4  | **2** | OK (effective W = 0.8) |
| **XFOOT** | 0.8 | 0.4  | **2** | OK |
| **XHDR**  | 0.8 | 0.4  | **2** | OK |
| XPDRVN    | 0.4 | 0.4  | 1 | OK |
| XFBN      | 0.5 | 0.5  | 1 | OK |
| **XODRVN**| 0.9 | 0.45 | **2** | OK |

Per-finger W chosen so that `2·rail_h + W_per_finger + 1.2 ≤ 2.59`,
with a small margin. Largest per-finger W = 0.75 (PMOS) and 0.5
(NMOS); both respect the budget.

## Final layout Y parameters

| Symbol | Value (µm) | Meaning |
|---|---|---|
| `rail_h` | 0.23 | M1 rail thickness (min width) |
| `Y_VPWR` | 4.81 .. 5.04 | VPWR rail (top, 0.23 µm thick) |
| `Y_VGND` | 0.00 .. 0.23 | VGND rail (bottom, 0.23 µm thick) |
| `cy_p` | 3.60 | PMOS row centerline |
| `cy_n` | 1.20 | NMOS row centerline (must be ≥ 0.625 + 0.5/2 + 0.23 = 1.105) |
| `nwell_bottom` | 2.20 | Top of p-well / bottom of n-well |
| `nwell_top` | 5.50 | N-well extends 0.46 µm above cell top (abutment) |
| `pwell_bottom` | −0.46 | P-well extends 0.46 µm below cell bottom |

**Verification of constraints with these numbers:**
- (a) Max PMOS W_p = 1.5 single OR per-finger 0.75 for `nf=2/4`:
  - cy_p + W_p/2 + 0.395 = 3.60 + 0.75 + 0.395 = 4.745 ≤ 4.81 − 0.23 = 4.58 ✗
  - **WAIT**: with single-finger W_p=1.5, this fails. With per-finger
    W_p=0.75 in `nf=2/4`: 3.60 + 0.375 + 0.395 = 4.37 ≤ 4.58 ✓.
  - For XPRE single W=1.0: 3.60 + 0.5 + 0.395 = 4.495 ≤ 4.58 ✓.
  - For XFBP/XCUTFB single W=0.5: 3.60 + 0.25 + 0.395 = 4.245 ✓.
- (b) Min NMOS gate-contact below rail: cy_n − W_n/2 − 0.395 − 0.23 =
  1.20 − 0.25 − 0.625 = 0.325 ≥ rail_h=0.23 ✓ (margin 0.095 µm).
- (c)+(d) N-well placement: with per-finger NMOS max W=0.5 single
  (XFBN), NMOS top = 1.20 + 0.25 = 1.45. DF.16: nwell_bottom ≥
  1.45 + 0.6 = 2.05. With per-finger PMOS max W=1.0 single (XPRE),
  PMOS bottom = 3.60 − 0.5 = 3.10. DF.4c: nwell_bottom ≤ 3.10 − 0.6
  = 2.50. We chose nwell_bottom = 2.20 ∈ [2.05, 2.50] ✓.

## Cell width estimate

Eleven SPICE-level FETs become **11 layout instances** (some with
nf > 1 internally). Per-instance widths (approximate, L=0.5/0.6):

| Type | Width estimate |
|---|---|
| `nf=1` PMOS / NMOS | ~1.4 µm |
| `nf=2` (any FET) | ~2.0 µm |
| `nf=4` (XPDRVP) | ~3.0 µm |

Cell width: dominated by PMOS row at ~1.4 (XPRE) + 1.4 (XFBP) + 1.4
(XCUTFB) + 3.0 (XPDRVP nf=4) + 2.0 (XODRVP nf=2) ≈ 9.2 µm + 5 gaps
at ≥ 0.36 µm DF.3a_MV spacing = ~11 µm. NMOS row similar or wider
due to 4 multi-finger devices. **Target cell width: ~12–14 µm**
(within the 15 µm guideline).

## Trade-off summary

Choosing **minimum-width rails (0.23 µm)** + **multi-finger via `nf`**
is the only path that keeps the original SPICE sizing (which the PVT
sweep validated) AND fits a 5.04 µm 9T 5V row. The trade-offs:

| Constraint | Cost |
|---|---|
| Rails narrower than stock 9T (which uses 0.46 µm thick rails) | Slightly higher rail IR drop; mitigated by parallel rails across cell rows at chip level |
| Multi-finger via `nf` | More FET instances per cell (~11 vs 12 single-finger); slightly larger cell width |
| Cell width ~12–14 µm vs stock inv at 2.24 µm | LSDL is 11-device topology vs CMOS inv's 2; intrinsic expansion |

## Re-derivation procedure (for future LSDL cells)

For a new LSDL cell `lsdl_<func>_<drive>`:

1. Read SPICE → list all FETs with their W and L.
2. Find the LARGEST W_p (max PMOS) and W_n (max NMOS).
3. If `W_p + W_n > 2.13 µm` (= 2.59 − 2·0.23): multi-finger the
   biggest. Per-finger W = ceil((2.13 − W_n)) for PMOS, etc.
4. `nf` for each FET = ceil(SPICE_W / per_finger_W).
5. Recompute cell width estimate. If > 15 µm: revisit the cell-row
   approach (consider double-height for that one cell only).
6. Use the Y parameters in this README (rails, cy_p, cy_n, nwell).

# Net-to-Terminal Routing Table (`lsdl_inv_x1`)

Reference table for every signal net's terminals and the chosen
routing strategy. Used during Phase D layout to ensure every M1 patch
is absorbed into the correct electrical net (not just any large M1
shape) — otherwise DRC may improve but LVS will fail.

**S/D contact x-offset convention (from FET center):**
- PMOS L=0.5, per-finger pitch 1.02 µm:
  - nf=1: contacts at ±1.02 — LEFT = source (chosen)
  - nf=2: contacts at -1.02, 0, +1.02 — OUTER = source, CENTER = drain
  - nf=4: contacts at ±2.04, ±1.02, 0 — pattern S-D-S-D-S (outer = source)
- NMOS L=0.6, per-finger pitch 1.12 µm:
  - nf=1: contacts at ±1.12 — LEFT = source (chosen)
  - nf=2: contacts at -1.12, 0, +1.12 — OUTER = source, CENTER = drain

**Gate-contact M1 patch x-positions (per finger):**
- nf=N FET: N fingers at centers `cx + (i - (N-1)/2) · pitch` for i ∈ [0..N-1]
- Each finger has 2 gate-contact M1 patches at x = `finger_center ± 0.21`
  (small lips around the polycontact)

**FET positions:**
- PMOS centers (slots 0..4): 1.5, 4.1, 6.7, 10.83, 15.47
- NMOS centers (slots 0..5): 1.5, 4.78, 8.06, 10.78, 12.94, 15.66

## Routing strategy rationale

- **Power nets (VPWR, VGND):** use **horizontal source sub-rails** in the
  per-row "source band" (NMOS sub-rail at y ≈ 0.65, PMOS sub-rail at y ≈
  4.0) connecting all same-net source caps in that row, then drop 1–2
  vertical trunks to the rail in **clear x positions** (not at finger
  pitches). Avoids the multi-finger gate-contact-patch conflict.
- **Internal-chain nets (cut_fb_src, hdr_src, foot_top):** SHORT local
  M1 between adjacent FETs only. No long-distance routing.
- **Gate nets (Clk, A, dyn, out_b, Out):** **M2 trunks** for any signal
  that crosses the cell. M1 only for short local doglegs from
  gate-contact patches to the M2 trunk.
- **Output (Out) pin:** widened M1 patch at the cell's right edge
  (x ≈ 17.0..17.2) connected via M2 trunk.
- **Input (A) pin:** widened M1 patch at the cell's left edge.

## Routing track scheme

Vertical Y positions (M1 horizontal tracks + M2 horizontal tracks):

```
y = 4.81 .. 5.04 : VPWR rail (M1)
y = 4.0  .. 4.05 : PMOS source sub-rail (M1, 0.23 µm thick)
y = 3.2  .. 3.5  : PMOS gate-contact patch row (gate-contact M1 lives here per device)
y = 2.5  .. 2.95 : MID-CELL TRACKS — 2 horizontal tracks available
                    Track A (y=2.5 .. 2.73): M2 trunk — out_b OR Clk
                    Track B (y=2.75 .. 2.98): M2 trunk — dyn OR Out
y = 2.0  .. 2.20 : NMOS gate-contact patch row
y = 0.65 .. 0.85 : NMOS bottom S/D cap row + source sub-rail (M1)
y = 0.00 .. 0.23 : VGND rail (M1)
```

M2 tracks in mid-cell carry the long-distance signal nets (Clk, dyn,
out_b, Out). The Clk net specifically benefits from M2 because it
fans out to both PMOS and NMOS rows (different y regions) without
fighting per-FET M1 patch geometry.

## DRC-priority for incremental routing

After each routed net, run KLayout DRC. Order of attention:
1. M1.2a (spacing) — most likely new failure mode when routing through
   dense regions
2. M1.3 (min area) — drops as nets get connected
3. via/contact rules
4. well/body rules (should already be clean)

## LVS-priority

After most routing exists:
1. **Device count first** — Magic extraction must report the same
   number of pfet_05v0 / nfet_05v0 as the SPICE. If `nf > 1` extracts
   as multiple separate FETs, fix via Netgen `--combine_devices` flag.
2. **Power and ground** — verify all PMOS bodies → VPWR, NMOS bodies
   → VGND, no VPWR-VGND short.
3. **Opens** — nets split unintentionally (e.g., a "dyn" island not
   connecting to all dyn pins).
4. **Shorts** — wrong-net merges (especially dyn↔out_b, dyn↔VPWR,
   out_b↔Out).
